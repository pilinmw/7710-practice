## T1: 封装类（验证 + 防御性副本 + static）

**何时用：** 题目要你写一个类，有字段、构造器验证、getter/setter、可能有计数器或统计方法。

```java
import java.util.*;

public class XXX {
    // 字段：不可变类型直接存；可变类型(List/Map)注意防御性副本；全类共享用 static
    private String name;
    private List<Integer> items;
    private static int count = 0;

    public XXX(String name) {
        // TODO: 1) null/非法验证（throw IllegalArgumentException）
        //       2) this.xxx = xxx 赋值
        //       3) 可变 field 用 new ArrayList<>() 初始化
        //       4) count++（放在验证之后！）
    }

    public String getName() {
        // TODO: String 不可变，直接 return
        return null;
    }

    public List<Integer> getItems() {
        // TODO: 返回防御性副本 new ArrayList<>(items)，不能 return items
        return null;
    }

    public void addItem(int x) {
        // TODO: 范围验证后 items.add(x)
    }

    public double average() {
        // TODO: 1) 空集合先返回 0.0（避免除零）
        //       2) 累加求和
        //       3) (double) 强制转换避免整数除法
        return 0.0;
    }

    public static int getCount() {
        // TODO: static 方法返回 static 字段
        return 0;
    }
}
```

> [!warning] T1 踩坑清单
> - [ ] `count++` 放在 null 检查**之后**（失败对象不该计数）
> - [ ] List/Map field 构造器里必须 `new` 初始化，否则 add 时 NPE
> - [ ] getter 返回 List/Date 等可变对象 → 必须 `new` 一份副本
> - [ ] 平均值/比例 → `(double)` 强制转换，否则整数除法砍小数
> - [ ] 空集合先判断，避免除零 / NPE

## T2: 继承 + 抽象类 + 多态

**何时用：** 题目有一个父类（常是 abstract），多个子类各自实现某方法，父类有个方法调用子类实现（多态）。

```java
// ===== 抽象父类 =====
public abstract class Shape {
    // TODO: 声明 abstract 方法（无方法体，分号结尾）
    public abstract double area();

    // concrete 方法：利用多态调用子类的 area()
    public String describe() {
        // TODO: 用 String.format("%.2f", area()) 拼字符串
        return null;
    }
}

// ===== 子类 1 =====
public class Circle extends Shape {
    private double radius;

    public Circle(double radius) {
        // TODO: 验证非负 + 赋值
    }

    @Override
    public double area() {
        // TODO: Math.PI * radius * radius
        return 0.0;
    }
}

// ===== 子类带父类字段时的写法（备用） =====
public class Sub extends Parent {
    private int extra;
    public Sub(int parentField, int extra) {
        super(parentField);   // 【关键】super(...) 必须是第一行
        this.extra = extra;
    }
}
```

> [!warning] T2 踩坑清单
> - [ ] abstract 方法没有 `{}`，直接分号结尾
> - [ ] abstract class 不能 `new`，但可以有 constructor（供子类 super 调用）
> - [ ] 子类必须实现所有 abstract 方法，否则子类也得是 abstract
> - [ ] `super(...)` 调父类构造器，必须是构造器第一条语句
> - [ ] `@Override` 加上，让编译器帮你检查方法名/签名
> - [ ] 多态：父类方法里调 `area()` 实际跑的是子类版本（Dynamic Binding）

## T3: 接口 + Comparator + 集合排序

**何时用：** 题目要你按某种规则排序，或多级排序（先按 A 再按 B），且常要求不修改原 list。

```java
import java.util.*;

public class XXXService {

    // 单键降序
    public List<Item> sortByValueDesc(List<Item> items) {
        // TODO: 1) 复制：new ArrayList<>(items)（不改原 list）
        //       2) copy.sort(Comparator.comparingDouble(Item::getValue).reversed())
        //       3) return copy
        return null;
    }

    // 多级排序：先按 name 升序，name 相同按 value 升序
    public List<Item> sortByNameThenValue(List<Item> items) {
        // TODO: copy.sort(
        //         Comparator.comparing(Item::getName)
        //                   .thenComparing(Item::getValue));
        return null;
    }
}
```

**Comparator 常用构造（记熟）：**
```java
Comparator.comparing(Item::getName)              // 按 String/Comparable 升序
Comparator.comparingInt(Item::getAge)            // 按 int
Comparator.comparingDouble(Item::getSalary)      // 按 double
        .reversed()                              // 反转（降序）
        .thenComparing(Item::getXxx)             // 第二排序键
```

> [!warning] T3 踩坑清单
> - [ ] 要求"不修改原 list" → 先 `new ArrayList<>(原)` 再 sort
> - [ ] 降序用 `.reversed()`，不要手写 `b - a`（可能溢出）
> - [ ] 多级排序用 `.thenComparing(...)`
> - [ ] `list.sort(cmp)` 或 `Collections.sort(list, cmp)` 都行
> - [ ] 方法引用 `Item::getName` 等价于 `x -> x.getName()`

```java
// Map 计数模式（备用，很常考）

```

> [!warning] T4 踩坑清单
> - [ ] 改了 equals **必须**改 hashCode（两者用相同字段）
> - [ ] equals 四步：this==o → null/类型检查 → 转型 → 比字段
> - [ ] `getClass() != o.getClass()` 比 instanceof 更严格（保证对称性）
> - [ ] 对象字段比较用 `Objects.equals()`（null 安全），不要直接 `.equals()`
> - [ ] hashCode 用 `Objects.hash(字段...)` 最省事
> - [ ] mutable 对象当 key 后修改 → hashCode 变 → 取不到（口试常问）

## T5: 泛型 + 通配符

**何时用：** 题目要写一个能装任意类型的容器，或一个能处理多种数字类型的方法。

```java
import java.util.*;

// 泛型类
public class Box<T> {
    private T value;

    public Box(T value) {
        // TODO: 赋值
    }

    public T get() { return null; }       // TODO
    public void set(T value) { }          // TODO
    public boolean isEmpty() {            // TODO: value == null
        return false;
    }
}

// 泛型工具方法
public class BoxUtils {

    // bounded wildcard：接受 List<Integer>、List<Double> 等
    public static double sumOfNumbers(List<? extends Number> nums) {
        // TODO: 遍历，累加 n.doubleValue()
        return 0.0;
    }

    // 泛型方法：<T> 写在返回类型前
    public static <T> T firstOrNull(Box<T> box) {
        // TODO: 非空返回 box.get()，否则 null
        return null;
    }
}
```

> [!warning] T5 踩坑清单
> - [ ] 泛型类：`class Box<T>`，字段/方法用 T
> - [ ] 泛型方法：`<T>` 写在返回类型**前面**：`public static <T> T foo(...)`
> - [ ] `<? extends Number>` = 接受 Number 及其子类（Integer/Double）
> - [ ] Number 用 `.doubleValue()` / `.intValue()` 取值
> - [ ] 创建：`new Box<>("x")`，菱形 `<>` 自动推断；空值要 `new Box<String>(null)`

```java
        //       return result;

```

**try/catch/finally 执行顺序（MCQ 必考）：**
```java
try {
    // 正常执行；遇异常立刻跳到 catch
} catch (XxxException e) {
    // 处理异常
} finally {
    // 【无论如何都执行】，即使 try/catch 里有 return 也先执行 finally
}
```

> [!warning] T6 踩坑清单
> - [ ] checked exception 继承 `Exception`；unchecked 继承 `RuntimeException`
> - [ ] checked 异常：方法签名要 `throws`，调用处要 try/catch 或继续 throws
> - [ ] 自定义异常构造器记得 `super(message)`
> - [ ] `Integer.parseInt(s)` 解析失败抛 `NumberFormatException`（unchecked）
> - [ ] finally 一定执行，即使前面有 return（return 值先算好，finally 后才真正返回）
> - [ ] "跳过非法项"用 try/catch 包住单次操作，catch 里啥都不做

## 通用 API 速查

```java
// ---- 集合创建 ----
List<Integer> list = new ArrayList<>();
List<Integer> copy = new ArrayList<>(list);        // 防御性副本
Set<String> set = new HashSet<>();

list.add(x); list.get(i); list.size(); list.isEmpty();

map.put(k, v); map.get(k); map.containsKey(k);
map.getOrDefault(k, 0);

set.add(x); set.contains(x);

list.sort(Comparator.comparing(X::getName));

Collections.sort(list);                            // 自然排序

// ---- 遍历 ----
```

## 通用踩坑总清单

> [!warning] 编译相关（编译错误 = 0 分！）
> - [ ] 括号配对：多/少一个 `}` 会让一片代码报错 → 点括号看配对
> - [ ] 方法必须 return 值（即使没实现，也 return null/0/false）
> - [ ] import 齐全（`java.util.*` 最省事）
> - [ ] 不改类名、方法签名、文件名（考试规定）

> [!warning] 逻辑相关
> - [ ] 整数除法：`int/int` 砍小数 → 要 `(double)`
> - [ ] 空集合：先判断 isEmpty，避免除零/NPE
> - [ ] 可变对象：getter 返回副本，别暴露内部引用
> - [ ] equals 改了 → hashCode 也要改
> - [ ] 排序不改原 list → 先复制
